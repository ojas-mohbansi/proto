Thesis

Most software waits. Proto doesn't.

Chatbots are a mirror — they reflect your questions back with answers. Proto is something different. It holds your goals in working memory, constructs an execution plan before you've finished your coffee, and moves — methodically, safely, with full auditability — while you're busy with everything else.

Built on hierarchical task network planning and local language models, Proto is an experiment in what software can be when it stops waiting to be told what to do. It is, at its core, a prototype for the kind of machine intelligence that earns its keep.

I
Process

How Proto thinks
and acts

01

Decompose the goal

Every intention you hand to Proto is broken apart by a Hierarchical Task Network planner — recursively divided into sub-tasks, and further into primitive actions that can be directly executed. Nothing is attempted as a whole. Everything is understood as parts first.

02

Consult memory & model

Before any action is taken, Proto reads its episodic memory — a living record of past outcomes, strategies, and context — and consults the language model for safety and intent alignment. This is the deliberation step. It does not rush.

03

Execute in a sandbox

Approved actions run in an isolated execution environment. Side effects are tracked and contained. A persistent watchdog monitors for failure and triggers autonomous recovery rather than silently dying.

04

Learn, then report

Every outcome — success, failure, retry — is committed to memory. Proto grows more effective at your tasks over time. Results surface on a live dashboard or directly in your WhatsApp, whichever you check first.

What is Proto?

Proto is a free, open-source proactive agent for developers. Unlike a chatbot or reactive assistant, Proto acts autonomously: it accepts a high-level goal, decomposes it into sub-tasks using hierarchical task network (HTN) planning, consults an episodic memory store and a local language model before each step, executes actions inside a sandboxed environment, and reports results — all without requiring the user to issue follow-up instructions.

Is Proto free?

Yes. Proto is completely free to use. It is released under the AGPL-3.0 open-source license. There are no subscription fees, no usage caps, and no paid tiers. The source code is publicly available on GitHub at https://github.com/ojas-mohbansi/proto.

Is Proto open source?

Yes. Proto is fully open source (FOSS). The entire codebase is licensed under the GNU Affero General Public License v3.0 (AGPL-3.0), which grants users the right to use, study, modify, and redistribute the software. Contributions and forks are welcome.

Where can I find Proto on GitHub?

Proto is hosted on GitHub at https://github.com/ojas-mohbansi/proto. The repository includes the full source code, a README with setup instructions, example .env configuration, and an issue tracker for bug reports and feature requests.

How does Proto work?

Proto uses hierarchical task network (HTN) planning — a classical AI planning technique — to recursively decompose a user-supplied goal into primitive executable actions. Before each action, Proto consults its episodic memory (a record of past outcomes) and a locally-hosted language model for safety and intent alignment. Approved actions run inside an isolated sandbox. A persistent watchdog monitors for failures and triggers autonomous recovery. Every outcome is committed back to memory, making Proto more effective at repeated tasks over time.

Does Proto require cloud services or send data externally?

No. Proto is designed to run entirely on local hardware. By default it connects to a locally-running LLM server such as Ollama. No data leaves the user's machine. Proto has zero cloud dependencies in its default configuration and collects no telemetry.

What language models does Proto support?

Proto supports any language model that exposes an OpenAI-compatible REST API. This includes locally-hosted models via Ollama (Llama 3, Mistral, Gemma, Qwen, and others), vllm, LM Studio, Groq, as well as cloud APIs such as OpenAI, Together AI, and Mistral AI. The active backend is configured via the OPENAI_BASE_URL environment variable.

What are the system requirements for Proto?

Proto requires Python 3.10 or later. It runs on Linux, macOS, and Windows. A locally-hosted language model (e.g. via Ollama) is recommended for fully private, offline use, but any OpenAI-compatible endpoint is supported.

How do I install Proto?

Clone the repository with `git clone https://github.com/ojas-mohbansi/proto`, install dependencies with `pip install -r requirements.txt`, copy `.env.example` to `.env` and configure your LLM endpoint, then run `python proto.py --goal "your goal here"`. An optional web dashboard can be launched with `python dashboard.py`.

What makes Proto different from other AI agents?

Most AI agents are reactive: they wait for a prompt and respond. Proto is proactive: it plans ahead, acts autonomously, self-corrects on failure, and learns from outcomes. Key differentiators include: (1) HTN-based goal decomposition rather than single-prompt generation, (2) episodic memory that persists across sessions, (3) a sandboxed execution environment with a watchdog for autonomous recovery, (4) full local execution with no mandatory cloud dependencies, and (5) an AGPL-3.0 license making it genuinely free and open source.
